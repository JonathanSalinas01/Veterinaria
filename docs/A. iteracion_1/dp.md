## Sistema de Gestión Veterinaria – Iteración 1: Development Plan (Sprint Backlog)

| **HU4 – Registro y Actualización de Dueños** | Como veterinario o asistente, quiero registrar y actualizar los datos personales de los dueños (nombre, DNI, teléfono, correo, dirección), para mantener actualizada la información de los clientes. | • Los datos se almacenan correctamente.<br>• Los campos obligatorios (Nombre, DNI, Teléfono) se validan antes de guardar. | • Diseñar modelo de datos Dueño (C#).<br>• Crear tabla Dueños en MySQL.<br>• Desarrollar formulario de Alta/Edición de Dueño (UI).<br>• Implementar lógica de negocio (CRUD) para Dueños.<br>• Validar que el DNI sea único.<br>• Pruebas unitarias para el servicio de Dueños. | Pareja A (Ana y Carlos) | Se prioriza esta HU para poder asociar mascotas en HU3. |
| **HU1 – Registro de Mascota** | Como veterinario, quiero registrar una nueva mascota con sus datos básicos (nombre, especie, raza, edad, sexo y peso), para mantener un control completo de los pacientes. | • Permite ingresar todos los datos obligatorios.<br>• El sistema valida que no existan campos vacíos. | • Diseñar modelo de datos Mascota (C#).<br>• Crear tabla Mascotas en MySQL.<br>• Desarrollar formulario de Alta de Mascota (UI).<br>• Implementar lógica de negocio (CRUD) para Mascotas.<br>• Probar inserción y validación de datos en la base de datos. | Pareja B (Juan y María) | El formulario de alta debe incluir la selección del dueño (HU3). |
| **HU3 – Asociación Mascota–Dueño** | Como veterinario, quiero vincular cada mascota con su respectivo dueño, para mantener la relación entre pacientes y sus propietarios. | • La mascota queda asociada a un dueño registrado.<br>• No se permite registrar mascotas sin dueño. | • Añadir clave foránea `id_dueño` en la tabla Mascotas.<br>• Modificar el formulario de alta de mascota para incluir un buscador/selector de Dueños (usando HU4).<br>• Pruebas de integridad referencial en base de datos. | Pareja B (Juan y María) | Se desarrolla en conjunto con HU1. |
| **HU11 – Autenticación de Usuario** | Como veterinario, quiero iniciar sesión con usuario y contraseña, para acceder de forma segura al sistema. | • El sistema valida credenciales correctamente.<br>• Se muestran mensajes de error si los datos son incorrectos.<br>• Las contraseñas se almacenan cifradas. | • Diseñar modelo de datos Usuario (C#).<br>• Crear tabla Usuarios (usuario, password_hash).<br>• Implementar lógica de cifrado (hashing) de contraseñas.<br>• Desarrollar vista de Login (UI).<br>• Implementar el servicio de autenticación.<br>• Crear usuario admin por defecto. | Pareja A (Ana y Carlos) | Se usará para restringir el acceso al sistema. |
